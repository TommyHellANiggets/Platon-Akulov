🚀 БЫСТРАЯ ИНСТРУКЦИЯ (3 ШАГА):
════════════════════════════════════════════════════════════════════════════════

ШАГ 1: ФИНАЛЬНАЯ ПОДГОТОВКА ЛОКАЛЬНО
─────────────────────────────────────
1. Откройте файл: passenger_wsgi.py
2. Замените YOUR_USERNAME на реальное имя пользователя от Reg.ru:

   SITE_ROOT = '/home/YOUR_USERNAME/public_html'
   VENV_PATH = '/home/YOUR_USERNAME/public_html/venv/lib/python3.10/site-packages'

3. Узнайте версию Python у Reg.ru и обновите версию в пути (3.8, 3.9, 3.10...)

ШАГ 2: ЗАГРУЗКА НА ХОСТИНГ (FTP)
─────────────────────────────────
1. Установите FTP клиент: FileZilla, WinSCP или Putty
2. Подключитесь:
   - Host: ftp.platon-akulov.ru
   - Username: ваше имя пользователя
   - Password: ваш пароль
   
3. Загрузите все файлы в: /public_html/
4. Убедитесь, что НЕ загружена папка venv/

ШАГ 3: НАСТРОЙКА НА СЕРВЕРЕ (SSH)
─────────────────────────────────
1. Подключитесь по SSH:
   ssh username@platon-akulov.ru

2. Выполните команды:
   cd ~/public_html
   python3 -m venv venv
   source venv/bin/activate
   pip install -r requirements.txt
   python manage.py migrate
   python manage.py collectstatic --noinput
   touch tmp/restart.txt

3. Готово! Откройте в браузере: https://platon-akulov.ru

✋ СТОП! ПРОЧИТАЙТЕ ЭТО:

   • Имя пользователя (username)
   • Версию Python на сервере

   • Доступен ли SSH

3. ⚠️  НЕ ЗАГРУЖАЙТЕ ПАПКУ VENV
   • Она создаст огромный архив (несколько сотен МБ)

   • Убедитесь, что db.sqlite3 загружена
   • На сервере выдайте права: chmod 666 db.sqlite3

  1. Откройте DEPLOYMENT_REGRU.md (раздел "Решение проблем")
  2. Проверьте логи: tail -f ~/public_html/passenger_wsgi.log
  3. Убедитесь, что passenger_wsgi.py обновлен правильно
  4. Обратитесь в поддержку Reg.ru

ЧАСТЫЕ ОШИБКИ:
  ❌ 500 Internal Server Error    → Проверьте passenger_wsgi.py пути
  ❌ Статика не загружается      → Запустите collectstatic
  ❌ "модуль django не найден"   → Убедитесь venv активирован
  ❌ БД не найдена               → Проверьте права доступа

💡 СОВЕТЫ:

  • Регулярно проверяйте размер БД (SQLite имеет ограничения)
  • Для production переведите DEBUG = False
  • Используйте более надежный SECRET_KEY в production

🎓 ДОПОЛНИТЕЛЬНО:
  • Оставьте DEBUG = True пока разбираетесь
  • Используйте логи для диагностики проблем
